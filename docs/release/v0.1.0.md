# v0.1.0 Release Notes Draft

## Summary

`v0.1.0` is the first usable XcaNet desktop release candidate.

It delivers a managed-default PKI workstation with secure local storage, certificate/CSR/CRL workflows, optional OpenSSL-assisted CSR signing, template-assisted issuance defaults, desktop import/export, and repeatable packaging support.

## Included in v0.1.0

- SQLite-backed local workspace with migrations
- master-password-based private-key protection at rest
- RSA and ECDSA key generation
- self-signed CA creation
- CSR creation and signing
- certificate parsing and detail inspection
- certificate revocation and CRL generation
- import/export for PEM, DER, PKCS#8, PKCS#12/PFX, PKCS#10 CSR, and CRLs
- optional OpenSSL bridge with managed fallback
- template CRUD and template-assisted issuance defaults
- desktop file dialogs, drag-and-drop import, diagnostics, and packaging scripts

## Important runtime behavior

- Managed crypto remains the default backend.
- OpenSSL is optional.
- Missing or invalid OpenSSL bridge artifacts do not block normal startup or managed workflows.

## Known limitations

- OpenSSL is not a full alternate backend yet; only the routed CSR-signing operation uses it.
- Templates provide practical defaults and validation, not enterprise policy-engine behavior.
- Packaged runtime verification is strongest on the current development platform; cross-platform packaging layout is validated through publish scripts and CI.

## Recommended operator guidance

- use managed-only mode unless OpenSSL-assisted CSR signing is specifically needed
- keep the master password safe; it protects stored private keys at rest
- check `Settings / Security` and startup logs first when diagnosing runtime issues
